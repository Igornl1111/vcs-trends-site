Key milestones in VCS history

Year System Type Significance
1972 SCCS (Source Code Control System) Centralized (Early) Developed at Bell Labs, it introduced the basic idea of tracking changes.
1982 RCS (Revision Control System) Local Automated change tracking for individual files using delta storage; lacked network support.
1986 CVS (Concurrent Versions System) Centralized Developed as a front-end to RCS, it introduced a client-server model and networked repositories for project collaboration.
2000 Git and Mercurial Centralized Successor to CVS that introduced atomic commits and better file handling, becoming highly popular by ~2006.
2005 Git and Mercurial Distributed (DVCS) Git was created by Linus Torvalds for the Linux kernel after its license for BitKeeper was revoked. DVCS became a major breakthrough.
2008 GitHub Platform/Hosting Launched a web-based collaboration platform on top of Git, introducing pull requests and greatly accelerating Git's adoption
2010s Wide Git Adoption Distributed Distributed version control became the new standard, with many projects and companies migrating from older systems (CVS, SVN) to Git.
2023 Large-Scale Growth Platform GitHub reported over 100 million developers using its platform
