What VCS is?


Version control systems (VCS) are essential tools that track changes in files (typically source code)

over time, allowing multiple people to collaborate, maintain a history of modifications, and revert

to earlier versions. The evolution of VCS has progressed from early centralized systems, which relied

on a single server, to modern distributed systems.

The core functions shared by all VCS tools

  • Repository: The database storing all versioned files and their history.
  • Working Copy: The local checkout of files you edit.
  • Commit: Saving a set of changes to the repository as a new revision.
  • Branch: A parallel line of development used to isolate work on features or fixes.
  • Merge: Integrating changes from one branch into another.

The key distinction is between:

  • Centralized Version Control (CVCS): Tools like RCS, CVS, and Subversion (SVN) use a single central repository. Collaboration requires network access to this server, which creates a single point of failure.
  • Distributed Version Control (DVCS): Tools like Git and Mercurial give every developer a full copy of the repository and its history. Developers can commit locally (even offline) and later push changes, providing greater flexibility, resilience, and faster operations. DVCS has become the dominant standard.
